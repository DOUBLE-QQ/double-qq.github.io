这两天被lantern莫名其妙的失效折磨的死去活来的，网速也时好时坏，让我萌生了自己搭建一个vpn的想法，看了些经验贴后感觉也不是特别困难，因此，今天就动手搭建了自己的vpn，运行结果果然爽的一比呀，网速杠杠的！在搭建中也遇到了一些问题，以此文做个总结。

第一步，选择vps服务商。

VPS，指Virtual Private Server 虚拟专用服务器，我们随后的操作就是利用这个境外服务器来作为我们的vpn，进而实现科学上网的目的。

看了好些经验贴，看到大多数人用的是vultr和搬瓦工这两个vps服务商。两者评价差不多，性价比也差不多，vultr评价似乎更高一些，因此我最终选了vultr，我选的套餐是：Linux centOS服务器，$5一个月，1000G流量，单核CPU，1G内存，25G的硬盘。我感觉是够用了，还有更便宜的$2.5一个月的。

补充一点，vultr在注册登陆后，会直接进入Billing页面，先让你给你账户充至少$10，支持支付宝、PayPal等，充值成功后才可以部署服务器，相关费用直接从你账户里扣，也算合理了，可以接受。

第二步，搭建服务器。

1. 选择机房

充值后就可以直接部署服务器了，会让你选择服务器地址（国家、城市之类），这时你可以到这个网页看下各地机房测试数据，里面有全国各个城市到vultr各机房网络速度的近期报告，你可以根据你的位置选择一个近期最快、最稳定安全的。我选择的机房是硅谷。

2. 选择服务器类型

选好机房后就选择服务器类别和大小（价钱），我是centOS 7 & $5

3. 额外特色

我只选了 Enable IPv6，其余为空

4. Startup Script

我未添加，为空

5. SSH keys

这里要添加一个SSH keys，作用是随后㐓通过ssh在本地连接上远程的服务器，然后可以在本地的终端上配置相关的参数。

首先，需要你在本地生成你设备的SSH key，我是mac，因此我介绍的是mac的操作步骤，Windows的可以看这个帮助文档：How do I generate my SSH keys?

  • 第一步，在你终端上run ssh-keygen -t rsa
  • 第二步，在系统询问你想把文件保存在哪儿时，直接按enter键跳过
  • 第三步，填入一个passphrase，任意英文就行，这个在之后ssh登陆服务器时会要求你输入，很重要。
  • 第四步，Run cat ~/.ssh/id_rsa.pub，你可以看到ssh key的内容，复制，随后会用
  • 第五步，复制上一步那个文件，以防ssh key丢失
  • 第六步，到vultr上https://my.vultr.com/sshkeys页面，选择添加SSH key （Add SSH Key）,然后键入一个名字，再把第四步复制的SSH Key粘贴，点击添加SSH Key,完成！

第三步，在服务器端配置shadowsocks server版本

如果你是mac可以直接通过终端连接vps，如果是Windows的话需要借助第三方工具，可以找下其他帖子。

首先，在你的vultr服务器页面复制你的服务器IP地址，然后打开你的终端，输入 ssh root@你的服务器IP地址，回车。

如果你的终端页面返回了一连串提示，如下图，那说明你的IP地址没问题，没有被墙，可以放心使用，你选择yes，然后会让你输入一个passphrase，就是配置SSH Key时你设置的，输入后你就会看到你已经和服务器连接成功了！

如果你的终端页面长时间没反应，然后最终返回的是 ssh: connect to host 128.199.244.xxx port 22: Operation timed out，十有八九你的IP被墙了，建议把这个服务器删掉重新再部署一个新的服务器，换个IP再试下，我换了第三个才成功。我整个过程在这一步卡了最长时间，因为之前一直连接失败后我没想到可能是IP本身问题，我一直以为是我配置问题，在那儿捣鼓了好长时间，最终才明白可能是IP本身问题，大家可以借鉴一下。

第四步，在本地配置shadowsocks

上一步完毕后，你就可以在本地终端配置你的服务器了，这样有两个好处：第一，vultr上虽然也会给你提供一个网页终端供你操作配置，但是很糟糕，我发现它上面的centOS服务器，在终端无法粘贴复制，也敲不出通配符"|",这样的话，随后的所有命令你都得一个一个的敲，非常非常累，因为后面代码中会有很多网址之类的，敲起来很麻烦，另外，没有 | 符号，更是闹心呀，命令都敲不全。第二，本地终端你自己应该都配置过一遍了，用起来更顺手、方便，粘贴复制什么也无障碍。所以还是推荐大家通过ssh 进行连接，然后在本地配置。

这一步，主要有三条命令，每次执行一条，主要目的是在你服务器上安装shadowsocks server版：

1
2
3

wget --no-check-certificate https://raw.githubusercontent.com/teddysun/shadowsocks_install/master/shadowsocks.sh
chmod +x shadowsocks.sh
./shadowsocks.sh 2>&1 | tee shadowsocks.log

执行过程中，系统会提示你手动输入一些参数，除了shadowsocks密码那一项，其他的你可以直接回车跳过去，这样那些参数就是默认值。

最后一条结束后，你会看到你服务器的各种信息，如下图，根据这些信息你也就可以直接配置你的vpn了！：

根据上面的信息，你可以根据自己设备类型，安装相应的shadowssocks 客户端，然后在服务器设置选项创建新的服务器，然后填入上面的ip地址、端口、密码等信息即可。

填写完毕后，你就可以科学上网了！！

第五步， TCP Fast Open

实际上只要具备上述四个信息，你就可以在自己的任意设备上进行登录使用了。但是为了更好的连接速度，你还需要多做几步。

首先是打开 TCP Fast Open，输入以下命令，意为用 nano 这个编辑器打开一个文件。

1

nano /etc/rc.local

你的「终端」会刷新一下，出现下图。

别慌张，它就是个文本编辑器。用方向键把光标移到最末端，粘贴下面这一行内容，然后按 Ctrl + X 退出。

1

echo 3 > /proc/sys/net/ipv4/tcp_fastopen

输入“Y”并回车确认退出。

然后依法炮制，输入：

1

nano /etc/sysctl.conf

在文末加上下面的内容，保存退出。

1

net.ipv4.tcp_fastopen = 3

再打开一个 Shadowsocks 配置文件。

1

nano /etc/shadowsocks.json

把其中 “fast_open” 一项的 false 替换成 true。

1

"fast_open":true

最后，输入以下命令重启 Shadowsocks。

1

/etc/init.d/shadowsocks restart

第六步 应用BBR加速器

在上面部署完毕vpn后，其实还可以通过锐速加速器这个软件来对你的vpn的网速进行优化。

TCP BBR 是 Google 于 2016 年发布的，一种避免网络拥塞的算法。目的是要尽量跑满带宽, 并且尽可能避免排队的情况。
实际使用下来，BBR 的速度提升效果并不比「锐速」（据说已倒闭）、kcptun（多倍发包耗流量）等差。而且 BBR 已植入 Linux 4.9+ 版本的内核中，服务器开启后就能有显著的效果提升。

安装 BBR

1
2
3
4

ssh root@<host>

wget --no-check-certificate https://github.com/teddysun/across/raw/master/bbr.sh && chmod +x bbr.sh && ./bbr.sh

按任意键开始安装，安装需要一段时间，等待一会即可。
安装完成后，脚本会提示需要重启 VPS，输入 y 并回车后重启。
重新使用 SSH 登录 VPS，这时 BBR 应该已经开启了。你可以使用以下两行命令验证一下。

1
2
3

uname -r
lsmod | grep bbr

如果输出的内核版本为 4.9 以上版本，且返回值有 tcp_bbr 模块的话，说明 bbr 已启动。
